\mbox{\hyperlink{class_test}{Test}} suite for \href{https://github.com/aseprite/aseprite}{\texttt{ Aseprite}} to avoid breaking backward compatibility.

This directory is cloned by the \href{https://github.com/aseprite/aseprite/blob/main/.github/workflows/build.yml}{\texttt{ build.\+yml}} action to run several automated tests after Aseprite is compiled\+:


\begin{DoxyItemize}
\item Save/load file formats correctly. For this we have {\ttfamily .aseprite}, {\ttfamily .png}, {\ttfamily .gif}, etc. files \href{https://github.com/aseprite/aseprite/tree/main/tests/sprites}{\texttt{ sprites}} folder.
\item \mbox{\hyperlink{class_test}{Test}} backward compatibility with \href{https://www.aseprite.org/docs/cli/}{\texttt{ Aseprite CLI}} options
\item Future \href{https://github.com/aseprite/api}{\texttt{ scripting API}} using \href{https://github.com/aseprite/aseprite/tree/main/tests/scripts}{\texttt{ scripts}}
\end{DoxyItemize}\hypertarget{md__ase_sprite_aseprite_tests__r_e_a_d_m_e_autotoc_md565}{}\doxysection{How to run tests?}\label{md__ase_sprite_aseprite_tests__r_e_a_d_m_e_autotoc_md565}
You have to set the {\ttfamily ASEPRITE} environment variable pointing to the Aseprite executable and then run {\ttfamily run-\/tests.\+sh} from Bash\+: \begin{DoxyVerb}export ASEPRITE=$HOME/your-aseprite-build/bin/aseprite
cd tests
bash run-tests.sh
\end{DoxyVerb}
 You can filter some tests with a regex giving a parameter to {\ttfamily run-\/tests.\+sh}, for example\+: \begin{DoxyVerb}run-tests.sh color
\end{DoxyVerb}
 Should run all tests which have the {\ttfamily color} word in their name.
