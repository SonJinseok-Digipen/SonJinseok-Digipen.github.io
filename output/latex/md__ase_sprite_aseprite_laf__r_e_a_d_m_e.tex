{\itshape laf} can be compiled with two back-\/ends ({\ttfamily LAF\+\_\+\+BACKEND})\+: {\ttfamily skia} or {\ttfamily none}.

When {\ttfamily LAF\+\_\+\+BACKEND=skia}, {\itshape laf} requires a compiled version of the \href{https://skia.org/}{\texttt{ Skia library}} from branch {\ttfamily aseprite-\/m96}. You can check the aseprite/skia fork which includes a \href{https://github.com/aseprite/skia/releases}{\texttt{ release with pre-\/built versions}}, or the check the \href{https://github.com/aseprite/skia\#readme}{\texttt{ instructions to compile skia}} from scratch.

When {\ttfamily LAF\+\_\+\+BACKEND=none}, the \href{http://www.pixman.org/}{\texttt{ Pixman library}} can be used as an alternative implementation of the {\ttfamily \mbox{\hyperlink{classgfx_1_1_region}{gfx\+::\+Region}}} class (generally if you\textquotesingle{}re using {\ttfamily laf-\/os} you will link it with Skia, so there is no need for Pixman at all).\hypertarget{md__ase_sprite_aseprite_laf__r_e_a_d_m_e_autotoc_md96}{}\doxysection{Compile}\label{md__ase_sprite_aseprite_laf__r_e_a_d_m_e_autotoc_md96}
To compile {\itshape laf} with Skia as backend you have to specify some variables pointing to a valid compiled version of Skia in your disk. In the following example {\ttfamily /skiadir} is the absolute path to a directory where the Skia source code + compiled version is, or just where you\textquotesingle{}ve uncompressed a pre-\/built package of Skia (note that in this case {\ttfamily /skiadir/out/\+Release-\/x64} should contain the Skia library files, i.\+e. {\ttfamily skia.\+lib} on Windows or {\ttfamily libskia.\+a} on other platforms)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https://github.com/aseprite/laf.git}
\DoxyCodeLine{cd\ laf}
\DoxyCodeLine{mkdir\ build}
\DoxyCodeLine{cd\ build}
\DoxyCodeLine{cmake\ -\/G\ Ninja\ \(\backslash\)}
\DoxyCodeLine{\ \ -\/DLAF\_BACKEND=skia\ \(\backslash\)}
\DoxyCodeLine{\ \ -\/DSKIA\_DIR=/skiadir\ \(\backslash\)}
\DoxyCodeLine{\ \ -\/DSKIA\_LIBRARY\_DIR=/skiadir/out/Release-\/x64\ \(\backslash\)}
\DoxyCodeLine{\ \ ..}
\DoxyCodeLine{ninja}
\DoxyCodeLine{./examples/helloworld}

\end{DoxyCode}


To compile only the library (without examples and tests) you can disable the {\ttfamily LAF\+\_\+\+WITH\+\_\+\+EXAMPLES}/{\ttfamily LAF\+\_\+\+WITH\+\_\+\+TESTS} options\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/DLAF\_WITH\_EXAMPLES=OFF\ -\/DLAF\_WITH\_TESTS=OFF\ ...}

\end{DoxyCode}
