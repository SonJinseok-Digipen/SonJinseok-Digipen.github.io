More info at \href{https://qoiformat.org}{\texttt{ https\+://qoiformat.\+org}}\hypertarget{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1807}{}\doxysubsection{Why?}\label{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1807}
Compared to stb\+\_\+image and stb\+\_\+image\+\_\+write QOI offers 20x-\/50x faster encoding, 3x-\/4x faster decoding and 20\% better compression. It\textquotesingle{}s also stupidly simple and fits in about 300 lines of C.\hypertarget{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1808}{}\doxysubsection{Example Usage}\label{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1808}

The recommended MIME type for QOI images is {\ttfamily image/qoi}. While QOI is not yet officially registered with IANA, I believe QOI has found enough adoption to prevent any future image format from choosing the same name, thus making a MIME type collision highly unlikely (\href{https://github.com/phoboslab/qoi/issues/167}{\texttt{ see \#167}}).

The recommended file extension for QOI images is {\ttfamily .qoi}\hypertarget{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1810}{}\doxysubsection{Limitations}\label{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1810}
The QOI file format allows for huge images with up to 18 exa-\/pixels. \mbox{\hyperlink{class_a}{A}} streaming en-\//decoder can handle these with minimal RAM requirements, assuming there is enough storage space.

This particular implementation of QOI however is limited to images with a maximum size of 400 million pixels. It will safely refuse to en-\//decode anything larger than that. This is not a streaming en-\//decoder. It loads the whole image file into RAM before doing any work and is not extensively optimized for performance (but it\textquotesingle{}s still very fast).

If this is a limitation for your use case, please look into any of the other implementations listed below.\hypertarget{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1811}{}\doxysubsection{Improvements, New Versions and Contributing}\label{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1811}
The QOI format has been finalized. It was a conscious decision to {\bfseries{not}} have a version number in the file header. If you have a working QOI implementation today, you can rest assured that it will be compatible with all QOI files tomorrow.

There are a lot of interesting ideas for a successor of QOI, but none of these will be implemented here. That doesn\textquotesingle{}t mean you shouldn\textquotesingle{}t experiment with QOI, but please be aware that pull requests that change the format will not be accepted.

Likewise, pull requests for performance improvements will probably not be accepted either, as this \char`\"{}reference implementation\char`\"{} tries to be as easy to read as possible.\hypertarget{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1812}{}\doxysubsection{Tools}\label{md__ase_sprite_aseprite_third_party_qoi__r_e_a_d_m_e_autotoc_md1812}
