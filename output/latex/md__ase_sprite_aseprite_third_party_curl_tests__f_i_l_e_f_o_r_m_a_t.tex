The curl test suite\textquotesingle{}s file format is very simple and extensible, closely resembling XML. All data for a single test case resides in a single ASCII file. Labels mark the beginning and the end of all sections, and each label must be written in its own line. Comments are either XML-\/style (enclosed with {\ttfamily \texorpdfstring{$<$}{<}!-\/-\/} and {\ttfamily -\/-\/\texorpdfstring{$>$}{>}}) or shell script style (beginning with {\ttfamily \#}) and must appear on their own lines and not alongside actual test data. Most test data files are syntactically valid XML, although a few files are not (lack of support for character entities and the preservation of CR/\+LF characters at the end of lines are the biggest differences).

Each test case source exists as a file matching the format {\ttfamily tests/data/test\+NUM}, where NUM is the unique test number, and must begin with a \textquotesingle{}testcase\textquotesingle{} tag, which encompasses the remainder of the file.\hypertarget{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1285}{}\doxysection{Preprocessing}\label{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1285}
When a test is to be executed, the source file is first preprocessed and variables are substituted by the their respective contents and the output version of the test file is stored as {\ttfamily log/test\+NUM}. That version is what will be read and used by the test servers.\hypertarget{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1286}{}\doxysubsection{Base64 Encoding}\label{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1286}
In the preprocess stage, a special instruction can be used to have runtests.\+pl base64 encode a certain section and insert in the generated output file. This is in particular good for test cases where the test tool is expected to pass in base64 encoded content that might use dynamic information that is unique for this particular test invocation, like the server port number.

To insert a base64 encoded string into the output, use this syntax\+: \begin{DoxyVerb}%b64[ data to encode ]b64%
\end{DoxyVerb}
 The data to encode can then use any of the existing variables mentioned below, or even percent-\/encoded individual bytes. As an example, insert the \mbox{\hyperlink{struct_h_t_t_p}{HTTP}} server\textquotesingle{}s port number (in ASCII) followed by a space and the hexadecimal byte 9a\+: \begin{DoxyVerb}%b64[%HTTPPORT %9a]b64%
\end{DoxyVerb}
 \hypertarget{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1287}{}\doxysubsection{Hexadecimal decoding}\label{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1287}
In the preprocess stage, a special instruction can be used to have runtests.\+pl generate a sequence of binary bytes.

To insert a sequence of bytes from a hex encoded string, use this syntax\+: \begin{DoxyVerb}%hex[ %XX-encoded data to decode ]hex%
\end{DoxyVerb}
 For example, to insert the binary octets 0, 1 and 255 into the test file\+: \begin{DoxyVerb}%hex[ %00%01%FF ]hex%
\end{DoxyVerb}
 \hypertarget{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1288}{}\doxysubsection{Repeat content}\label{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1288}
In the preprocess stage, a special instruction can be used to have runtests.\+pl generate a repetetive sequence of bytes.

To insert a sequence of repeat bytes, use this syntax to make the {\ttfamily \texorpdfstring{$<$}{<}string\texorpdfstring{$>$}{>}} get repeated {\ttfamily \texorpdfstring{$<$}{<}number\texorpdfstring{$>$}{>}} of times. The number has to be 1 or large and the string may contain {\ttfamily HH} hexadecimal codes\+: \begin{DoxyVerb}%repeat[<number> x <string>]%
\end{DoxyVerb}
 For example, to insert the word hello a 100 times\+: \begin{DoxyVerb}%repeat[100 x hello]%
\end{DoxyVerb}
 \hypertarget{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1289}{}\doxysubsection{Conditional lines}\label{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1289}
Lines in the test file can be made to appear conditionally on a specific feature (see the \char`\"{}features\char`\"{} section below) being set or not set. If the specific feature is present, the following lines will be output, otherwise it outputs nothing, until a following else or endif clause. Like this\+: \begin{DoxyVerb}%if brotli
Accept-Encoding
%endif
\end{DoxyVerb}
 It can also check for the inversed condition, so if the feature us {\itshape not} set by the use of an exclamation mark\+: \begin{DoxyVerb}%if !brotli
Accept-Encoding: not-brotli
%endif
\end{DoxyVerb}
 You can also make an \char`\"{}else\char`\"{} clause to get output for the opposite condition, like\+: \begin{DoxyVerb}%if brotli
Accept-Encoding: brotli
%else
Accept-Encoding: nothing
%endif
\end{DoxyVerb}

You can only do one conditional at a time and you can only check for a single feature in it.\hypertarget{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1290}{}\doxysection{Variables}\label{md__ase_sprite_aseprite_third_party_curl_tests__f_i_l_e_f_o_r_m_a_t_autotoc_md1290}
