This is the curl man page generator. It generates a single nroff man page output from the set of sources files in this directory.

There is one source file for each supported command line option. The output gets {\ttfamily page-\/header} prepended and {\ttfamily page-\/footer} appended. The format is described below.\hypertarget{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md741}{}\doxysection{Option files}\label{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md741}
Each command line option is described in a file named {\ttfamily \texorpdfstring{$<$}{<}long name\texorpdfstring{$>$}{>}.d}, where option name is written without any prefixing dashes. Like the file name for the -\/v, --verbose option is named {\ttfamily verbose.\+d}.

Each file has a set of meta-\/data and a body of text.\hypertarget{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md742}{}\doxysubsection{Meta-\/data}\label{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md742}
\begin{DoxyVerb}Short: (single letter, without dash)
Long: (long form name, without dashes)
Arg: (the argument the option takes)
Magic: (description of "magic" options)
Tags: (space separated list)
Protocols: (space separated list for which protocols this option works)
Added: (version number in which this was added)
Mutexed: (space separated list of options this overrides, no dashes)
Requires: (space separated list of features this requires, no dashes)
See-also: (space separated list of related options, no dashes)
Help: (short text for the --help output for this option)
Example: (example command line, without "curl" and can use `$URL`)
--- (end of meta-data)
\end{DoxyVerb}
 \hypertarget{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md743}{}\doxysubsection{Body}\label{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md743}
The body of the description. Only refer to options with their long form option version, like {\ttfamily -\/-\/verbose}. The output generator will replace such with the correct markup that shows both short and long version.

Text written within {\ttfamily $\ast$asterisks$\ast$} will get shown using italics. Text within two {\ttfamily $\ast$$\ast$asterisks$\ast$$\ast$} will get shown using bold.\hypertarget{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md744}{}\doxysection{Header and footer}\label{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md744}
{\ttfamily page-\/header} is the file that will be output before the generated options output for the master man page.

{\ttfamily page-\/footer} is appended after all the individual options.\hypertarget{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md745}{}\doxysection{Generate}\label{md__ase_sprite_aseprite_third_party_curl_docs_cmdline_opts__m_a_n_p_a_g_e_autotoc_md745}
{\ttfamily ./gen.pl mainpage}

This command outputs a single huge nroff file, meant to become {\ttfamily curl.\+1}. The full curl man page.

{\ttfamily ./gen.pl listhelp}

Generates a full {\ttfamily curl -\/-\/help} output for all known command line options.
