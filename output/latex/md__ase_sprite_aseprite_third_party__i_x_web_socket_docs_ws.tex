\hypertarget{md__ase_sprite_aseprite_third_party__i_x_web_socket_docs_ws_autotoc_md1764}{}\doxysection{Websocket proxy}\label{md__ase_sprite_aseprite_third_party__i_x_web_socket_docs_ws_autotoc_md1764}

\begin{DoxyCode}{0}
\DoxyCodeLine{ws\ proxy\_server\ -\/-\/remote\_host\ ws://127.0.0.1:9000\ -\/v}
\DoxyCodeLine{Listening\ on\ 127.0.0.1:8008}

\end{DoxyCode}


If you connect to ws\+://127.0.\+0.\+1\+:8008, the proxy will connect to ws\+://127.0.\+0.\+1\+:9000 and pass all traffic to this server.

You can also use a more complex setup if you want to redirect to different websocket servers based on the hostname your client is trying to connect to. If you have multiple CNAME aliases that point to the same server.

\mbox{\hyperlink{class_a}{A}} JSON config file is used to express that mapping ; here connecting to echo.\+jeanserge.\+com will proxy the client to ws\+://localhost\+:8008 on the local machine (which actually runs ws echo\+\_\+server), while connecting to bavarde.\+jeanserge.\+com will proxy the client to ws\+://localhost\+:5678 where a cobra python server is running. As a side note you will need a wildcard SSL certificate if you want to have SSL enabled on that machine.


\begin{DoxyCode}{0}
\DoxyCodeLine{echo.jeanserge.com=ws://localhost:8008}
\DoxyCodeLine{bavarde.jeanserge.com=ws://localhost:5678}

\end{DoxyCode}
 The --config\+\_\+path option is required to instruct ws proxy\+\_\+server to read that file.

\DoxyCodeLine{ws\ proxy\_server\ -\/-\/config\_path\ proxyConfig.json\ -\/-\/port\ 8765}
